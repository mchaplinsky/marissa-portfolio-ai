\f0\fs24 \cf0 # Research to Design Workflow\
\
Marissa Chaplinsky integrates research directly into the design process so that product decisions are grounded in real user insights rather than assumptions.\
\
Research does not exist as a separate phase but instead informs each stage of design development.\
\
---\
\
## Understanding the Problem Space\
\
Before proposing solutions, Marissa focuses on understanding the environment in which users operate.\
\
Research activities may include:\
\
\'95 user interviews  \
\'95 analytics reviews  \
\'95 support ticket analysis  \
\'95 stakeholder workshops\
\
These inputs help reveal the root causes behind user challenges.\
\
---\
\
## Synthesizing Research Insights\
\
Raw research data must be translated into meaningful insights. Marissa organizes research findings through synthesis techniques such as:\
\
\'95 affinity mapping  \
\'95 pattern identification  \
\'95 journey mapping\
\
This process helps teams identify recurring themes in user behavior.\
\
---\
\
## Defining Opportunities\
\
Once patterns emerge from the research, opportunities for improvement begin to surface.\
\
Marissa works with teams to define design opportunities such as:\
\
\'95 simplifying workflows  \
\'95 improving navigation structures  \
\'95 reducing friction in key tasks\
\
These opportunities guide concept exploration.\
\
---\
\
## Translating Insights Into Design\
\
Research insights inform the creation of design concepts and prototypes.\
\
Design artifacts may include:\
\
\'95 user flows  \
\'95 wireframes  \
\'95 interaction models  \
\'95 system diagrams\
\
Each artifact reflects insights gathered during the research process.\
\
---\
\
## Validating Through Testing\
\
Design concepts are tested with users to ensure that solutions actually address the identified problems.\
\
Feedback from testing informs further refinements and ensures that design decisions remain aligned with real user needs.\
